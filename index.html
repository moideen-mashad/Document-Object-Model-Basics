The DOM Tree Explained

A visual and interactive guide to understanding the Document Object Model in JavaScript.

1. What is the DOM?

The Document Object Model (DOM) is an interface that browsers use to turn HTML into a structured tree of objects (nodes). This allows programming languages like JavaScript to easily read, add, change, or delete content on the web page dynamically.

When you write HTML, you are writing text. When the browser loads it, it translates that text into a living tree structure in its memory.

2. The Folder Analogy

Think of the DOM tree exactly like the folder and file structure on your computer.

🗄️
<html> Element

Your root folder (e.g., C:\ drive).

📁
<body> Element

A main system folder inside your root.

📂
<div> & <ul>

Sub-folders inside the body.

📄
Text & Attributes

Files sitting inside those sub-folders.

3. JavaScript Code Interaction

Because the DOM is structured as a tree, we can use JavaScript to "climb" it, find specific elements, and manipulate them.

// Finding a specific element const itemList = document.getElementById('item-list'); // Creating a new element const newItem = document.createElement('li'); newItem.textContent = 'Item 3 (Added via JS)'; // Adding it to the DOM itemList.appendChild(newItem);
